【博客事记】停用七牛云图床，更换图片链接

2018年10月28日
邮箱里发来一份邮件：测试域名回收通知–七牛云服务，一直没在意。今天发现之前的博客里的图片全部失效了，当然这也无可厚非，毕竟免费的服务，说没就没了呗。于是我想下载回七牛云上的图片，放到GitHub Page 里就行了，其实根本不需要什么第三方图床嘛。 但是恶心的时，竟然连下载也不行。为了恢复之前的图片，新建一个空间，就有有了一个测试域名，但是只有30天有效期，所以就等于没什么用处了。 qshell 使用官方提供的工具：qshell，可以紧急恢复一下。 列出 leebolgpic 空间中的图片到 move.conf： ./qshell listbucket leebolgpic move.conf 将 leebolgpic 空间的图片移动到 fuckq： ./qshell batchmove -force -overwrite leebolgpic fuckq move.conf 有趣的是，我们刚刚下载的move.conf文件，并不能直接用户移动，里面有很多垃圾信息，刚好昨天看了一下vim的宏录制功能可以派上用场： 根据 qdown.conf 的配置下载文件： ./qshell qdownload 10 qdown.conf 1 {"dest_dir" : "/home/leer/Downloads/q", "cdn_domain": "phac5ew5s.bkt.clouddn.com", "bucket" : "fuckq", "suffixes" : ".png, .jpg, .jpeg, .gif, .ico" } 接下来就是手动复制粘贴，一个一个替换链接了/(ㄒoㄒ)/~~

Concurrent with Semaphore

2018年10月18日
| 自习室

Semaphore

Dijkstra 大佬提出了一种解决不同线程之前同步和互斥问题的方法。这种方法就是 PV 操作，它基于 Semaphore（信号量）。

可以把信号量简单看成一个非负整数，只能使用两种操作来改变它的值，这两种操作就是 P 和 V，用伪代码描述如下（与教材《操作系统教程》上不同，这里的信号量不能为负值）。PV 操作的伪代码实现如下：

  • P(s)

    P(s) {
    if (p != 0) {
     s--;
    } else {
     append this thread to list;
     sleep();
    }
}

  • V(s)

    V(s) {
    s++;
    if (list is no empty) {
        list.pop().wakeup();
    }
}

P 和 V 的执行过程都是不可打断的，并且 P 和 V 要成对的出现。这样就保证了程序不可能进入信号量为负值的状态，可以利用这个特性实现进程之间的互斥和同步。

6.828 Homework 2:Shell

2018年9月29日
| 自习室

这次的作业是实现一个简单的 shell。通过阅读 xv6 book 的第零章，基本上就可以实现题目中所说的三个功能，也是shell的基本功能：运行简单的命令，输入输出重定向，管道。

题目提供的sh.c已经实现了命令的解析（大概看了一下流程，大概就是个递归下降解释器吧，这次作业无需关心这个），只需要完成runcmd函数中缺少的代码即可。

流程

先从 main 函数看起：

int main(void) {
  static char buf[100];
  int fd, r;

  // Read and run input commands.
  while (getcmd(buf, sizeof(buf)) >= 0) {
    if (buf[0] == 'c' && buf[1] == 'd' && buf[2] == ' ') {
      // Clumsy but will have to do for now.
      // Chdir has no effect on the parent if run in the child.
      buf[strlen(buf) - 1] = 0; // chop \n
      if (chdir(buf + 3) < 0)
        fprintf(stderr, "cannot cd %s\n", buf + 3);
      continue;
    }
    if (fork1() == 0)
      runcmd(parsecmd(buf));
    wait(&r);
  }
  exit(0);
}

main 函数循环调用 getcmd从 console 读取命令，对 cd 命令做了特别处理。之后fork创建子进程，调用parsecmd解析命令，再调用runcmd执行得到的命令。最后调用wait等待子进程结束。main 的基本流程就这样。

| 碎碎念
跟人聊天，该人懂一点中文，外派几年管理江苏的一家企业，聊着聊着忽然说：“我发现在中国的日常生活里，人们喜欢彼此制造困难让大家日子难过（Die Leute mögen sich gegenseitig das Leben schwer machen.)而这么做通常都没什么理由或好处——这是我特别不能理解的一点。” - 罗开Kai 事情还得从今年（2018）的 4 月 10 号开始，大概就是同学觉得我会 “做网站”，就把我的联系方式给 F 了。F 开始的时候倒是很客气的，现在看来都是假惺惺。 最开始，F 这样问我：“你好，同学，我们这边需要一个会数据库或者网页制作的同学，希望加入我们的公司，这个团队也可以做到毕业以后。我们现在主要做的是人工穿戴智能，你在这边，主要做的就是制作相应的网页或者建立数据库，这样的。酬劳一类的，我们可以见面聊一下”。我就以为他是需要一个管理数据库的 DBA，就说我会数据库，网站只会一点。 4 月 10 号晚上，见面的时候，我话比较少，但是 F 给我的感觉还是不错的，当时很热情。期间不时给我画大饼，说他这个项目怎么怎么好，待遇怎么怎么好。我问：“已经做出产品来了吗？”F 开始吱吱唔唔，开始转移话题，用一句草草带过：“已经在做了。”我想还是这样啊，没有一个靠谱的。然后 F 开始介绍他的公司，说要给我股份。我没有给出明确的答复，因为让我感觉不靠谱，而且如果拿了股份有一种被束缚的感觉。 下一次见面的时候，他拿来了大概是分股份的合同，我没签，我说：“我可以拿工资帮你干活，不要股份”。F 说好。过了几天，他又拿着一份劳动合同找我签字。至于酬劳一字未提，当然这个时候我还是相信 F 的，我也知道刚开始创业是有困难。而且他也没有那种老板的架子，比较亲切，至少在这个时候。（虽然我很反感这样的，明明不太熟还那么亲切，但也没有到反胃的程度吧。 我开始了解到 F 所说的 “已经在做了”，其实并无此事。他们什么都没有，要从零开始，但是「只差一个程序员」。 到了 4 月 22 号。让我到他的办公室来，我问他：办公室在哪？他说：“在寝室，辅导员的办公室”。我一想为什么骗我？为什么和一开始说的不一样。如果情况是这样，可以告诉我真实的情况，为什么要骗我？这时候就开始不信任了吧。F 开始给我分配一些任务。一开始是是说要做桌面端的一个项目，然后我发现就我一个程序员。。大概就是「只差一个程序员」系列了吧。我想了一下桌面端的我知道 Java swing 和 Qt，接下来的一天我就开始了解这方面的。但是 F 下一天给我看了一下其他人做的成品，我当时有点生气，我说：“这不是网站吗？” F：“对啊，就是网站，就是做一个网站”。无语。。。我的一天就这样浪费了。 我估计这个网站也是比较简单，所以还是答应了。如果数据接口都有的话，但是事实不是如此。我问 F 有没有这个项目的文档，F 说：“没有，要不我帮你写一份吧。“所以说到底还是一个 Idea？为什么一开始要和我扯那些犊子。这更让我觉得这是一个「只差一个程序员」的项目。这时候我已經对这个公司不报希望了，想着做好这个项目，拿了这个项目的钱就走人。 F 的那个”办公室“我只去过一次，就在也不去了。他说那是他们团队工作的地方。最让我觉得不可思议的地方是，F 竟然在工作的时候抽烟，虽然他问了我抽不抽，我说不抽，他就出去抽了。还是无法相信竟然在办公的时候抽烟，这里不是说不能抽烟，你抽不抽烟和我没关系，但是至少工作的时候还是不要抽吧？？*这时候我对 F 个人也不报希望了，甚至开始讨厌。*我似乎同时想到了这个项目的下场。 接下来的发展更加戏剧性，F 开始给我一系列的 Excle 表格，我也认了，因为这事我自己也干过，然后就转化一下格式，简化一些数据就导入数据库了。这里面的工作量，唉，我还不如去搬砖。最后，经过漫长的 ”wtf wtf wtf…“ 数据是有了（一部分）。接下来 F 开始疯狂提要求，我说没有数据我没有办法的。最后他说去和数据提供方沟通沟通。
